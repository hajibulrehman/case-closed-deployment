# Deploying to Render

## Prerequisites
- A [Render](https://render.com) account
- The repo pushed to GitHub or GitLab

---

## Steps

### 1. Push the repo to GitHub
```bash
git add .
git commit -m "chore: prepare for Render deployment"
git push
```

### 2. Create services via render.yaml (Blueprint)
1. Go to **Render Dashboard → New → Blueprint**
2. Connect your GitHub repo
3. Render will detect `render.yaml` and create:
   - `case-closed-db` — free PostgreSQL database
   - `case-closed-backend` — Node/Express API
   - `case-closed-frontend` — React static site

### 3. Set environment variables
After the initial deploy, set these in the Render dashboard:

**Backend service → Environment:**
| Key | Value |
|-----|-------|
| `DATABASE_URL` | *(auto-filled from the linked DB)* |
| `JWT_SECRET` | *(auto-generated by Render)* |
| `CORS_ORIGIN` | `https://case-closed-frontend.onrender.com` *(your actual frontend URL)* |

**Frontend service → Environment:**
| Key | Value |
|-----|-------|
| `VITE_API_URL` | `https://case-closed-backend.onrender.com` *(your actual backend URL)* |

> After setting `VITE_API_URL`, **trigger a manual redeploy** of the frontend so Vite picks it up at build time.

### 4. Run the database migration
The backend's build command (`npm run build`) runs `prisma generate && prisma migrate deploy` automatically on every deploy.

### 5. (Optional) Seed data
After the first deploy, open the Render **Shell** tab on the backend service and run:
```bash
node src/seed.js
```

---

## Notes on uploaded files
The `backend/uploads/` folder is **ephemeral** on Render's free tier — files are wiped on redeploy. For a personal project this is usually acceptable. If you need persistent uploads, consider:
- Render persistent disk (paid)
- Cloudflare R2 (free tier, S3-compatible)
- Supabase Storage (free tier)

## Local development (unchanged)
```bash
# Backend
cd backend
npm install
npm run dev

# Frontend (separate terminal)
cd frontend
npm install
npm run dev
```
